kubsh — собственная командная оболочка на C++
kubsh — это легковесная UNIX‑подобная оболочка, реализованная на C++ с нуля. Проект демонстрирует глубокое понимание системного программирования, управления процессами, работы с файловой системой, сигналами и интеграции с системными утилитами.

Основные возможности
Встроенные команды:

echo / debug – вывод текста (с поддержкой кавычек).

\e $VAR – вывод переменной окружения (например, \e $PATH выводит пути по одному на строку).

\l /dev/sdX – вывод информации о разделах диска (через lsblk).

exit / \q – завершение оболочки.

Виртуальная файловая система (VFS) для пользователей:

При запуске создаётся директория ~/users (в тестовой среде – /opt/users).

Для каждого пользователя, способного логиниться (с /bin/bash или /bin/sh), создаётся подкаталог с файлами:

id – UID пользователя

home – путь к домашней директории

shell – путь к оболочке

Создание нового подкаталога вызывает adduser, удаление – userdel.

История команд:
Все введённые команды сохраняются в ~/.kubsh_history.

Обработка сигналов:
При получении SIGHUP оболочка выводит "Configuration reloaded".

Выполнение внешних команд:
Поддерживается запуск любых исполняемых файлов с поиском в $PATH.

Поддержка тестирования:
Режим test mode (с VFS в /opt/users) используется в CI для автоматизированного тестирования.

Сборка и установка
make          # компиляция
make deb      # сборка deb-пакета
sudo dpkg -i kubsh.deb   # установка
После установки оболочка запускается командой kubsh.

Тестирование и CI
Проект настроен на автоматическую сборку и тестирование через GitHub Actions.
Используются pytest и Docker-контейнер с тестовой средой.
Все базовые тесты (1–8) проходят успешно, тест №9 (создание пользователя через VFS) исправлен добавлением синхронизации и задержки для устранения race condition.

Технологии
Язык: C++17

Системные вызовы: fork, execvp, waitpid, signal, sigaction, inotify, getpwnam, getenv

Сборка: Make, deb-пакетирование

Тестирование: pytest, Docker, GitHub Actions

Платформа: Linux (Debian/Ubuntu)
